# 伴伴 Android APK 测试指南

## 测试环境要求

### 真机测试
- Android 8.0+ (API 26+)
- 麦克风权限已授权
- 网络连接正常

### 模拟器测试
- Android Studio Emulator
- 系统镜像: Android 11/12/13
- 启用麦克风模拟

---

## 手动测试步骤

### 1. 安装
```bash
adb install -r build/app/outputs/flutter-apk/app-release.apk
```

### 2. 功能测试

#### 权限测试
- [ ] 首次启动请求麦克风权限
- [ ] 拒绝权限后显示提示
- [ ] 同意权限后进入主界面

#### 语音交互测试
- [ ] 长按麦克风按钮 → 显示红色录音指示器
- [ ] 说话 3-5 秒后松手 → 停止录音
- [ ] 显示转写文字 → 发送成功
- [ ] 收到伴伴回复 → TTS 播放正常

#### 文字交互测试
- [ ] 输入框输入文字
- [ ] 点击发送按钮
- [ ] 消息显示在列表中
- [ ] 收到伴伴回复

#### 边界测试
- [ ] 长按后立即取消（滑动取消）
- [ ] 录音时间少于 500ms
- [ ] 网络断开时发送消息
- [ ] 后台返回前台恢复连接

---

## 常见问题排查

### 麦克风无响应
1. 检查应用权限: 设置 → 应用 → 伴伴 → 权限 → 麦克风
2. 检查是否被其他应用占用
3. 重启应用重试

### 录音无法停止
1. 检查是否有错误日志: `adb logcat | grep flutter`
2. 强制停止应用后重试

### TTS 无声音
1. 检查媒体音量
2. 检查是否静音模式
3. 查看后端 TTS 服务是否正常
